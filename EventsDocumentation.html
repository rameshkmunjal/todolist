ToDo List Events Document

** Events for ToDo List functionality

Socket Connection Management

  • CONNECTION
    Event Triggered when a socket connection request is initiated from frontend

  • DISCONNECT
    Event Handler for graceful socket disconnection

Task Management

  • SET-USER
    User is set upon socket connection and his details like userId and fullName are stored.

  • SEND-LIST-DETAILS-TO-ITEM-BOX
    Details of list are sent to the item component to fetch items of list with a given Id .

  • SEND-ITEM-DETAILS-TO-SUB-ITEM-BOX
    Details of item are sent to the sub-item component to fetch sub-items of an item with a given Id .

  • GET-ALL-LISTS
    All list of a userId are fetched.

  • ITEMS-BY-LIST-ID
    All Items of a list with given Id are fetched

  • SUB-ITEMS-BY-ITEM-ID
    All sub-Items of an item with given Id are fetched

  • DELETE-TASK
    A common socket event is emitted for deleting a list or item or sub-item . Data carries the type like list , item or sub-item along details and on that basis function is called to delete .

  • CREATE-TASK
    A common socket event is emitted for creating a list or item or sub-item . Data carries the type like list , item or sub-item along details and on that basis function is called to create .

  • EDIT-TASK
    A common socket event is emitted for editing a list or item or sub-item . Data carries the type like list , item or sub-item and on that basis function is called to edit.

  • CHANGE-STATUS
    A common socket event is emitted for changing status of a list or item or sub-item . Data carries the type like list , item or sub-item along details and on that basis function is called to change the status .

UNDO functionality

  • UNDO-DELETE
    A common socket event is emitted for undo previous action of deleting a list or item or sub-item . Data carries the type like list , item or sub-item along details and on that basis function is called to undo.
  • UNDO-CREATE
    A common socket event is emitted for undo previous action of creating a list or item or sub-item . Data carries the type like list , item or sub-item along details and on that basis function is called to undo.
  • UNDO-EDIT
    A common socket event is emitted for undo previous action of editing a list or item or sub-item . Data carries the type like list , item or sub-item along details and on that basis function is called to undo.

Multi-User management

  • SEND-FRIEND-REQUEST
    This event is emitted to send friend request to other user. Both users id and name are sent. Handler function emits an event in response to display message on screen of other user.

  • SEND-USER-DETAILS
    As a user logs in - his details like userId and fullName are sent to the child components by home component. And when a friend name is selected - his details are sent to home component with same purpose.

Notifications Management

  • SEND-CURRENT-NOTIFICATION
    Notification data is sent to socket side . Handler calls a function to input details in notification document in database .

  • SHOW-ALL-NOTIFICATIONS
    All notifications are demanded from socket side by emitting this event.
